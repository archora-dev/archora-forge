# Release 1.3.0

## Summary

Forge 1.3.0 tightens the paid-pilot and pull-request review workflow. The main change is not a new framework target; it is clearer evidence for deciding whether an OpenAPI change should merge and whether a team should adopt generated output.

## Changed

- PR impact comments now include merge decision, merge risk, reason, source usage and next actions.
- `check` and `audit` reports include a `pass`, `warn` or `fail` gate and a recommended CI mode.
- GitHub Actions generation supports comment-only and block-merge modes through `--gate comment` and `--gate block`.
- `pilot` writes a reviewer package with `pilot-report.md`, `impact-pr.md`, `impact.md`, `impact.json`, audit artifacts and `go-no-go.md`.
- HTML reports render readiness gate details and reviewer checklists.
- Public Forge demo/proof artifacts were refreshed from the current CLI output.

## OpenAPI Hardening

- Added ugly OpenAPI fixtures for:
  - `oneOf`, `allOf` and discriminator-heavy schemas;
  - cookie API key auth and operation-level security;
  - binary upload/download and unsupported content types;
  - nullable fields and `type: [string, "null"]`;
  - unsafe schema names, operation IDs, path params and reserved property names;
  - header parameters.
- The hardening tests verify partial useful output, generated TypeScript plan coverage and explicit diagnostics for unsupported cases.

## Documentation

- Added a team rollout guide for trial branch, first PR, advisory CI, blocking CI and adoption decision.
- Expanded paid pilot packaging with inputs, outputs, generated `forge-pilot` folder contents and success/no-go criteria.
- Added honest comparisons with OpenAPI Generator, Orval and openapi-typescript.
- Documented `readiness.gate.result` in the API stability policy.
- Updated public demo package and site proof fixtures.

## Compatibility

- Version updated from `1.2.2` to `1.3.0`.
- CLI JSON top-level fields remain stable.
- New readiness fields are additive:
  - `readiness.gate.result`;
  - `readiness.gate.recommendedCiMode`;
  - `readiness.gate.reason`;
  - `readiness.reviewerChecklist`.
- Generated file layout remains framework-neutral TypeScript.

## Required Checks

Before publishing:

```bash
pnpm release:check
```

In `archora-site`:

```bash
pnpm verify
```

## Publish Notes

- Commit Forge changes before tagging.
- Commit `archora-site` fixture refresh separately if the site is released from its own repository.
- Run package verification from the final workspace state before `npm publish`.
