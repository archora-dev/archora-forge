# Archora Forge v1.0.0

Archora Forge v1 is the first public-preview/private-beta release target for local-first OpenAPI to typed frontend resource generation.

## Highlights

- Stable CLI JSON report surfaces for `inspect`, `validate`, `lint`, `generate --dry-run`, `diff` and `check`.
- HTML report output for validation, inspection and CI check workflows.
- Adoption report sections for readiness, generator metadata and schema coverage.
- Contract diff changelog summaries for release notes and pull-request review.
- Committed curated real-world fixture suite for CI regression coverage.
- Typed generated clients, query keys, operation helpers, schema-derived model types, resource metadata, permissions, i18n and mocks.
- Runtime auth presets for bearer, OAuth2 bearer-token usage and header API keys.
- Runtime retries, timeouts, tracing hooks, query array serialization and typed `ForgeHttpError` bodies.
- CI adoption kit documentation with strict/advisory modes, artifact policy and pull-request workflow.
- Config presets for feature-sliced, simple and multi-schema repository layouts.
- Support for common binary, text, form, wildcard response and empty-response OpenAPI patterns.
- Stable v1 docs for API stability, generated file layout, private schema CI and multi-schema monorepos.

## Supported Scope

- OpenAPI 3.x JSON/YAML from local files or remote HTTP(S) URLs.
- Framework-neutral TypeScript output.
- Common REST resource patterns plus explicit non-CRUD operation helpers.
- CI drift checks and report artifacts.
- Zod and Valibot validation artifacts for common JSON schema shapes as opt-in generation modes.

## Known Limits

- This release is not positioned as production-ready.
- Framework components are intentionally out of scope.
- Discriminator-based polymorphism remains diagnostic-only.
- Complete OpenAPI serialization coverage is not claimed.
- OAuth token acquisition and refresh remain application responsibilities.
- Generated output should be committed and reviewed; Forge is not a hosted registry or SaaS workflow.
- Zod and Valibot validation artifacts are experimental opt-in modes.
- TanStack-style usage requires a consuming-app integration wrapper in this preview.
