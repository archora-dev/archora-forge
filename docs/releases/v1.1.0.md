# Archora Forge v1.1.0

Archora Forge v1.1.0 turns the v1 generator into a stronger self-serve adoption product: teams can inspect impact, review readiness and attach generated evidence without a sales-assisted walkthrough.

## Highlights

- Self-serve audit package with HTML, Markdown, JSON, CI summary, adoption plan and generated preview artifacts.
- Resource Explorer in audit reports for quick review of generated resources, operations, hooks, mocks and diagnostics.
- Schema Coverage Matrix in `check --json` and audit reports for generated, diagnostic-only and unsupported operation counts.
- Frontend impact reports for pull-request review, including generated output impact and repo usage scanning.
- Contract diff changelog formatting for release notes and schema review.
- Start Guide and package-readiness docs for faster first-run evaluation.
- Public demo audit and impact artifacts for buyer-facing documentation.
- CodeQL security hardening for path-template parsing without regex backtracking risk.
- Regression coverage for report output, public API contract stability and generated example fixtures.

## Compatibility

- This is a minor release with no intentional breaking CLI command removals.
- `check --json` now includes additive `coverage` data at the top level and per schema.
- Generated metadata now reports Forge `1.1.0`; projects with committed generated output should regenerate before relying on drift checks.
- Existing config presets, generated file layout and runtime package names remain stable.

## Required Verification

```bash
pnpm release:check
```
