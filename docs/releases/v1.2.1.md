# Archora Forge v1.2.1

Archora Forge v1.2.1 turns the paid-pilot workflow into a first-class local product path: teams can request a license, review API impact against a git base ref, generate a pilot package and wire the same workflow into GitHub Actions.

## Highlights

- `license request` generates a safe trial/pilot license request file without schemas, source code, env vars or private absolute paths.
- `impact --base <ref>` compares the current schema against a git base ref without preparing a separate old schema file.
- `pilot --base <ref>` creates a one-folder paid-pilot package with impact, audit, PR comment and go/no-go artifacts.
- `demo --out <path>` creates a self-contained local impact demo with sample schemas, source usage and reports.
- `explain <diagnostic-code>` and `explain --list` turn diagnostics into adoption guidance.
- `ci init github --schema <path> --base <ref> --mode impact|pilot` generates GitHub workflow templates for impact or pilot review.
- Commercial license activation now has a local signed-key flow and clearer docs for manual trial/pilot requests.
- Docs now position Forge around frontend API impact before merge, not generic code generation.

## Compatibility

- This is a patch release over the unpublished-in-repo `1.2.0` attempt. The npm `1.2.0` core package had incorrect `workspace:` metadata; use `1.2.1`.
- `impact` keeps the two-schema form and adds the one-schema `--base <ref>` form.
- `pilot` accepts either `--old <schema>` or `--base <ref>`.
- Generated metadata now reports Forge `1.2.1`; projects with committed generated output should regenerate before relying on drift checks.
- Existing config presets, generated file layout and runtime package names remain stable.
