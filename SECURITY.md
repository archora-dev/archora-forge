# Security Policy

Archora Forge is a local-first generator. Do not commit API tokens, schema registry credentials or generated files containing secrets.

## Reporting

For security-sensitive reports, do not open public discussions or PRs. Email **akotov@archora.dev** with the subject prefix `[security]`, or contact **@akotofff** on Telegram. Do not disclose publicly until a fix is released.

## Current Scope

Supported auth behavior is limited to runtime header presets. OAuth refresh flows, token storage and enterprise auth policy are not implemented.
